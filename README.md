# 拼豆图纸转换器

打开 `index.html` 即可使用，不需要安装依赖。

## 功能

- 上传图片并转换为拼豆图纸
- 默认 `50×50` 和“原图匹配”，适合先看可拼效果
- 支持 `30×30` 到 `200×200` 多档预设和自定义规格
- 支持 `64×64 儿童画` 低像素推荐预设
- 支持居中裁剪、完整留边、拉伸填满
- 支持透明 PNG、接近纯色背景、灰白棋盘格假透明背景的主体裁剪与空白边缘识别
- 支持原图匹配、轻度简化、儿童画简化、卡通块面、轮廓强化
- 使用 LAB 色差匹配色号
- 可限制最大颜色数
- 默认开启低像素特征保护，下降到 `50×50` 时优先保留眼睛、嘴、装饰点等小面积高对比特征
- 儿童画、卡通等风格使用边缘保持平滑，避免普通模糊把五官和轮廓抹掉
- 可开启 Floyd-Steinberg 抖动
- 可通过邻域清理减少孤立杂色
- 儿童画模式会保守合并色块，默认不强加深色轮廓线
- 图纸格内可显示色号
- 图纸预览可缩放、适应窗口、触控板捏合缩放和按住拖动平移，不影响 PNG/SVG 导出尺寸
- 预览使用高清 backing canvas，PNG 默认 2× 导出，也可选 1× 到 4×
- 每 25 格默认显示分块粗线
- 颜色用量会显示套餐盒号 `f`、`g`、`h`、`i`、`j`、`k`
- 导出 PNG、SVG、用量 CSV；外侧空白不计入用豆数

## 色号说明

色号库来自当前文件夹中的 `截屏2026-05-16 下午4.07.23.png`，包含套餐 `f`、`g`、`h`、`i`、`j`、`k` 共 144 个色号。

`palette.js` 中的 HEX 是从截图色块采样得到的近似值，用于图片匹配和屏幕预览；如果后续有官方 RGB/HEX 色值，直接替换 `palette.js` 中对应色号的 `hex` 即可。

## 使用建议

- 普通头像或小图案：先用默认 `50×50` + “原图匹配”
- 小五官、小装饰点丢失时：保持“低像素特征保护”开启
- 希望低像素但高辨识度：先用 `64×64 儿童画`
- 细节较多的图片：用 `100×100`
- 制作实物优先从“原图匹配”开始，再改“轻度简化”或“儿童画简化”
- 五官、线条、外轮廓容易丢时，用“轮廓强化”
- 渐变图：开启抖动，但实物会更碎
- 购买清单：看页面里的盒号列或下载用量 CSV，估算包数按每包 1000 颗计算

## 儿童画简化逻辑

儿童画模式的目标不是重新生成陌生画风，而是在尽量保留原图特征的前提下减少摆豆难度：

1. 默认使用 24 色，不再把颜色压得过少
2. 先做低像素特征保护，保留小五官和装饰点
3. 使用边缘保持平滑，只合并相近颜色，不跨轮廓平均
4. 默认不强加深色描边，只有提高轮廓强度时才加线稿
5. 轻量邻域清理，减少孤立杂色但不抹掉主要细节

## 发布到 GitHub Pages

这是纯静态项目，发布源可以直接使用 `main` 分支的根目录。

```bash
git init
git add .
git commit -m "Add bead pattern converter"
git branch -M main
git remote add origin https://github.com/<你的用户名>/<仓库名>.git
git push -u origin main
```

然后在 GitHub 仓库中进入 `Settings` → `Pages`：

- Source 选择 `Deploy from a branch`
- Branch 选择 `main`
- Folder 选择 `/ (root)`

项目已包含 `.nojekyll`，GitHub Pages 会按普通静态文件发布，不走 Jekyll 处理。

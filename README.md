# 拼豆图纸转换器

打开 `index.html` 即可使用，不需要安装依赖。

## 功能

- 上传图片并转换为拼豆图纸
- 支持 `50×50`、`100×100`、自定义规格
- 支持 `64×64 儿童画` 低像素推荐预设
- 支持居中裁剪、完整留边、拉伸填满
- 支持透明 PNG 或接近纯色背景的主体裁剪
- 支持儿童画简化、拼豆简化、卡通块面、轮廓强化、原图匹配
- 使用 LAB 色差匹配色号
- 可限制最大颜色数
- 可开启 Floyd-Steinberg 抖动
- 可通过邻域清理减少孤立杂色
- 儿童画模式会强制大色块、少色号、深色轮廓线
- 图纸格内可显示色号
- 每 25 格默认显示分块粗线
- 导出 PNG、SVG、用量 CSV

## 色号说明

色号库来自当前文件夹中的 `截屏2026-05-16 下午4.07.23.png`，包含套餐 `f`、`g`、`h`、`i`、`j`、`k` 共 144 个色号。

`palette.js` 中的 HEX 是从截图色块采样得到的近似值，用于图片匹配和屏幕预览；如果后续有官方 RGB/HEX 色值，直接替换 `palette.js` 中对应色号的 `hex` 即可。

## 使用建议

- 普通头像或小图案：先用 `50×50`
- 希望低像素但高辨识度：先用 `64×64 儿童画`
- 细节较多的图片：用 `100×100`
- 制作实物优先用“儿童画简化”或“拼豆简化”
- 五官、线条、外轮廓容易丢时，用“轮廓强化”
- 渐变图：开启抖动，但实物会更碎
- 购买清单：下载用量 CSV，估算包数按每包 1000 颗计算

## 儿童画简化逻辑

儿童画模式的目标不是还原照片，而是减少摆豆难度：

1. 自动把默认规格降到 `64×64`
2. 默认把最大颜色数压到 18 色
3. 多轮平滑，去掉皮肤、毛发、背景里的细碎噪点
4. 把颜色压成更大的块面
5. 单独检测高对比边缘，用深色豆保留外轮廓和五官线条
6. 多轮邻域清理，减少一颗一颗的孤立杂色

## 发布到 GitHub Pages

这是纯静态项目，发布源可以直接使用 `main` 分支的根目录。

```bash
git init
git add .
git commit -m "Add bead pattern converter"
git branch -M main
git remote add origin https://github.com/<你的用户名>/<仓库名>.git
git push -u origin main
```

然后在 GitHub 仓库中进入 `Settings` → `Pages`：

- Source 选择 `Deploy from a branch`
- Branch 选择 `main`
- Folder 选择 `/ (root)`

项目已包含 `.nojekyll`，GitHub Pages 会按普通静态文件发布，不走 Jekyll 处理。
